Notes on Becoming a Udacity Code Reviewer Course

Revision Process

1 - Read Read Read:

All project documentation from Udacity is available in this GitHub repository. It's necessary to read this information to know all project specification.

Next step is to read documentation sent from the student.

2 - Scan and Run:

First step is to scan student code to see how it is structured.

Next step is to run the program to see if it works as expected.

3 - Provide Review:

Two ways of revision:

  1. Code review: line-by-line commentary on the code, highlighting what about it is awesome and what can be improved.
  2. Project review: use each project grading criteria and classify the project items where it Exceeds Specifications, Meets Specifications or Does Not Meet Specifications.

SPECS ideas

Simplify Code

It must be analyzed if the student code can be simplified somehow.

Practice Kindness

It's important to remember that a student took hours of his life watching the videos and preparing the projects. So, any effort to reduce frustration must be taken. One way to do so is to be kind with them.

In each revision, the revisor must find some characteristics of the code to mark as "Awesome".

Explain Why & With Examples

Each time a correction is done, it's necessary to explain why, preferably pointing to documentation like Google Guides or Python Library Docs, for example. The revisor can also write some code examples to the student. Gist is a good place to put these examples.

Connect Learners

Suggest to the student some useful source of knowledge like Tweeter, e-mail groups, etc.

Style Consistently

The student can use any style he wishes (eg: Python Style Guide), but the chosen style must be used throughout all the code.
